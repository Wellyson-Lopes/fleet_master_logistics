FleetMaster — Design System
Referência para IA · v1.0 · React Native + NativeWind 4
Projeto: FleetMaster Logistics — App Mobile do Motorista (módulo separado do dashboard web de gestão).
Escopo de dados: Todo o app é centrado no motorista autenticado. Não existem listas de "todos os motoristas", "toda a frota" ou "todas as viagens da empresa" — cada tela mostra apenas viagens, ocorrências, abastecimentos, veículo e documentos do usuário logado (autenticação via JWT).
App Mobile: React Native 0.74+ · NativeWind 4.x · flowbite-react-native · React Navigation 6
Fontes: Inter (UI principal) · JetBrains Mono (placas, IDs, código)
Paleta base: Navy/Blue/Teal/Amber — fundo escuro nos headers, fundo claro nas telas de conteúdo.
Público: Exclusivamente motoristas (acesso via convite, JWT). Gestores usam o dashboard web separado.
Idioma: Português Brasil. Datas: DD/MM/AAAA. Moeda: R$ 0.000,00. Placa: AAA-0000.

• Link "← NomeTela" em 11px text-white/50 acima do título

• Icon buttons 34×34px bg-white/10 rounded-[10px]

Segmented Control:
• bg-slate-100 · rounded-xl · p-1
• Tab ativo: bg-white rounded-[9px] shadow-sm
• Margem: mx-[14px] my-[10px]
Padrão de Telas
Toda tela segue a estrutura vertical:

StatusBar (fixo, navy)
AppHeader (fixo, navy)
SearchBar? (opcional, sticky)
SegmentedTabs? (opcional)
ScrollView (flex:1, overflow)
TabBar (fixo, bottom)

Scroll acontece apenas no ScrollView. Header e tab bar nunca scrollam. KeyboardAvoidingView envolve forms.

• Gesture de swipe-back sempre habilitado
• Transição padrão: slide horizontal (iOS) / fade (Android)
